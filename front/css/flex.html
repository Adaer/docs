                          # flex解答

                          Flex 布局，可以简便、完整、响应式地实现各种页面布局。 Flex 是 Flexible Box 的缩写，意为"弹性布局"，用来为盒状模型提供最大的灵活性。

                          • 容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。
                          • 主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end。
                          • 交叉轴的开始位置叫做cross start，结束位置叫做cross end。
                          • 项目默认沿主轴排列。
                          • 单个项目占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

                          核心

                          flex是一种线性/轴线对称的布局

                          # flex-grow

                          子项目的放大比例，默认为0，即如果容器存在剩余空间，也不放大。
                          当容器有剩余空间时，如果所有子项目的flex-grow为1，其中一个为2，那么为2的子项目将会占据更多的空间。

                          剩余空间计算公式：

                          1. 计算剩余空间: M = 容器宽度 - 所有子项目宽度
                          2. 计算总等分: T = 子项目1的flex-grow + 子项目n的flex-grow
                          3. 计算宽度: 子项目1宽度 = (子项目1的flex-grow / T) * M

                          # flex-shrink

                          子项目的缩小比例，默认为1，即如果容器的宽度小于所有子项目宽度的总和时，将会缩小子项目。
                          当容器有宽度不够时，如果所有子项目的flex-shrink为1，其中一个为0，那么为0的子项目的宽度将不变，为1的宽度将会缩小

                          收缩计算公式：

                          1. 计算溢出空间: M = 所有子项目宽度 - 容器宽度
                          2. 计算收缩比例:
                            • 子项目1的flex-basis基准值 * 子项目1的flex-shrink值 * p + 子项目n的flex-basis基准值 * 子项目n的flex-shrink值 * p = M
                            • p = M / 子项目1的flex-basis基准值 * 子项目1的flex-shrink值 + 子项目n的flex-basis基准值 * 子项目n的flex-shrink值
                          3. 计算宽度 子项目1的flex-basis基准值 * (1 - 子项目1的flex-shrink值 * p)

                          # flex-basis

                          子项目的基准值，默认为auto，即浏览器自动计算子项目的宽度，可以直接设置一个宽度（比如100px）
